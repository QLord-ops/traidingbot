# Binance Intraday Bot v0.5 — веб-панель + Testnet MVP + Telegram

## Что это

Локальная система для тестирования интрадей-стратегии на Binance USDⓈ-M Futures:

- **Backtest** по публичным историческим свечам (многомесячная история с кэшем);
- **Testnet engine** — исполнение сигналов на Binance Demo Trading
  (`demo-fapi.binance.com`) с обязательными SL/TP и лимитами риска;
- **Реальная торговля отсутствует и заблокирована на уровне кода.**

## Возможности v0.4

- диапазон дат, пагинация истории и локальный кэш свечей (`data/cache/`);
- комиссии maker/taker, проскальзывание, **funding fees** по реальной истории ставок;
- консервативная обработка свечи с одновременным SL/TP и гэпов сквозь стоп;
- дневные лимиты в backtest: максимум 3 сделки/день, дневной стоп-лосс 1.5%;
- equity/drawdown графики, месячная доходность, expectancy, средний R,
  максимальная серия убытков;
- **walk-forward** — проверка устойчивости по независимым окнам;
- профили параметров и сравнение прогонов;
- Testnet-панель: запуск/остановка engine, позиции, журнал событий,
  аварийное закрытие (STOP);
- предупреждения о переоптимизации в интерфейсе;
- **Telegram (v0.5)**: уведомления об открытии/закрытии позиций, аварийных
  событиях и ошибках; команды `/status` и аварийная `/stop` (`/panic`) —
  только из чата владельца (`TELEGRAM_CHAT_ID`).

## Запуск через Docker (рекомендуется)

```bash
docker compose up --build
```

Панель: http://127.0.0.1:8000 · Healthcheck: http://127.0.0.1:8000/health

## Локальный запуск (Python 3.12+)

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
# source .venv/bin/activate
pip install -r requirements.txt
python run_web.py
```

CLI-backtest без панели:

```bash
python -m src.run_backtest --symbol BTCUSDT --days 365 --walk-forward 4
```

Тесты:

```bash
python -m pytest tests -v
```

## Настройка Testnet (Demo Trading)

1. Получите API key/secret в Binance Futures Demo Trading.
2. Скопируйте `.env.example` в `.env`, заполните `BINANCE_API_KEY`,
   `BINANCE_API_SECRET`, установите `TRADING_MODE=testnet`.
3. Откройте вкладку «Testnet» в панели и нажмите «Запустить».

Гарантии безопасности engine:

- работает **только** с `demo-fapi.binance.com` — с боевым API он отказывается
  стартовать;
- позиция без подтверждённого биржей Stop Loss немедленно закрывается;
- идемпотентные `clientOrderId`: перезапуск не открывает дублирующую позицию;
- reconciliation при старте: состояние восстанавливается с биржи;
- дневной лимит убытка блокирует новые входы до следующего дня (UTC);
- кнопка STOP отменяет все ордера и закрывает все позиции.

## Безопасность

- `.env` не коммитится (`.gitignore`); секреты не отображаются в интерфейсе.
- API-ключ создавайте **без права вывода средств** и с IP-allowlist.
- `ENABLE_LIVE_ORDERS=true` приводит к ошибке конфигурации — live-режим
  отсутствует в v0.4 и будет обсуждаться только после 100–200 задокументированных
  Testnet-сделок с приемлемой статистикой.
- Пользователь в Германии: до любого разговора о live-режиме требуется проверка
  фактической и юридической доступности Binance Futures для аккаунта.
  Обход региональных ограничений не поддерживается.

## Важное про результаты backtest

Ни один результат backtest не является обещанием прибыли. Малое число сделок,
высокий Profit Factor на коротком периоде и «идеальная» кривая баланса — признаки
переоптимизации. Доверять можно только статистике, устойчивой на out-of-sample
периодах и walk-forward, после комиссий, funding и проскальзывания.

## Настройка Telegram (опционально)

1. Создайте бота через @BotFather, получите токен.
2. Напишите боту любое сообщение, узнайте свой chat id (например, через
   `https://api.telegram.org/bot<токен>/getUpdates`).
3. Заполните `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` в `.env`.

Команды из чужих чатов игнорируются. Сбой Telegram не влияет на торговый цикл.
