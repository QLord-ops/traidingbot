Работай как senior Python quant/backend engineer и разработчик безопасных торговых систем. Не доверяй существующему прототипу автоматически: сначала проведи аудит архитектуры, backtest, расчёта риска, Binance API client и веб-интерфейса. Главная цель ближайшего этапа: получить воспроизводимый веб-интерфейс для качественного backtest и безопасный Binance USDⓈ-M Futures Testnet MVP. Реальные сделки и live mode должны оставаться жёстко заблокированными.

Выполни работу в таком порядке:

1. Инициализируй Git при необходимости, проверь remote и подключи `https://github.com/QLord-ops/traidingbot.git`.
2. Удали из проекта кэш, логи и генерируемые файлы; проверь `.gitignore`; никогда не коммить секреты.
3. Установи зависимости, запусти pytest и веб-приложение.
4. Проведи аудит и исправь критические ошибки.
5. Сделай запуск одной командой через Docker Compose и также оставь понятный локальный Python-запуск.
6. Улучши backtest: исторические данные с пагинацией и кэшем, диапазон дат, комиссии maker/taker, funding, проскальзывание/spread, консервативная обработка одновременного SL/TP, equity/drawdown curves, monthly returns, expectancy, average R, losing streaks, out-of-sample и walk-forward. Строго исключи look-ahead bias.
7. Улучши русскую веб-панель: профили параметров, диапазон дат, график баланса и просадки, график сделок, сравнение наборов параметров, экспорт.
8. После устойчивого backtest начни Testnet engine: reconciliation, idempotency, precision filters, isolated margin, leverage <=3, max 1 position, risk 0.25%, daily loss 1.5%, max 3 trades/day, обязательные защитные SL/TP, emergency close, restart recovery.
