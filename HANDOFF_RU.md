# Передача проекта Claude Code

## Контекст

Пользователь хочет создать интрадей-бота для торговли фьючерсами через Binance с веб-интерфейсом. Конечная цель — управляемая из браузера система с backtest, Testnet и только после доказанной работоспособности — реальная торговля малым капиталом.

Репозиторий пользователя:
`https://github.com/QLord-ops/traidingbot.git`

Название репозитория написано `traidingbot` — не исправлять без просьбы пользователя.

## Уже согласованная конфигурация

- Рынок: Binance USDⓈ-M Futures.
- Стиль: intraday.
- Основные пары на первом этапе: BTCUSDT и ETHUSDT.
- Сигнальный таймфрейм: 15m.
- Фильтр направления: 1h.
- Margin mode: isolated.
- Начальное плечо: максимум 3x.
- Риск на сделку: 0.25% баланса.
- Дневной лимит убытка: 1.5%.
- Максимум одна открытая позиция.
- Максимум 3 сделки в день.
- Обязательные Stop Loss и Take Profit.
- Запрещены мартингейл, сетка, неограниченное усреднение и увеличение риска после убытка.
- Реальная торговля должна быть заблокирована по умолчанию.

## Начальная гипотеза стратегии

Тренд определяется на 1h:

- LONG: цена выше EMA200 и EMA20 выше EMA50.
- SHORT: цена ниже EMA200 и EMA20 ниже EMA50.

Вход на 15m:

- EMA20/EMA50 подтверждают направление;
- пробой high/low предыдущей закрытой свечи;
- объём выше среднего объёма за период, умноженного на коэффициент;
- минимальная волатильность проверяется через ATR/close;
- сигнал оценивается score-системой;
- вход выполняется только на следующей свече после сигнала, чтобы исключить look-ahead bias.

Stop Loss рассчитывается по локальной структуре и ATR. Take Profit задаётся через Reward/Risk, начальное значение 1.8.

Это только тестируемая гипотеза, а не доказанная прибыльная стратегия.

## Что уже реализовано

Проект на Python содержит:

- минимальный Binance USDⓈ-M Futures REST client;
- загрузку klines;
- EMA, ATR и фильтр объёма;
- генерацию LONG / SHORT / HOLD;
- risk-based position sizing;
- торговые правила `tickSize`, `stepSize`, min quantity и min notional;
- backtest с комиссией и проскальзыванием;
- SQLite-журнал;
- FastAPI веб-панель;
- выгрузку CSV и JSON;
- pytest-тесты.

Код является прототипом и требует аудита. Нельзя считать методы реального исполнения готовыми к деньгам без проверки актуальной документации Binance.

## Важные ограничения

1. Пользователь находится в Германии. До реальной торговли обязательно проверить, законно и фактически ли Binance Futures доступны его аккаунту. Не помогать обходить ограничения через VPN, чужой KYC или другую юрисдикцию.
2. API secret никогда не коммитить и не показывать в интерфейсе.
3. API key должен быть без права вывода средств и с IP allowlist.
4. Реальный режим должен требовать многоступенчатого включения и отдельного подтверждения.
5. В случае ошибки постановки SL/TP позиция должна немедленно закрываться или переходить в аварийное состояние — нельзя оставлять незащищённую позицию.
6. Не обещать прибыль. Все выводы только по статистике после комиссий, funding и realistic execution assumptions.

## Следующая задача

Продолжить с версии веб-панели и довести проект до безопасного Testnet MVP.

Приоритетный порядок:

1. Полный аудит текущего кода и исправление архитектурных ошибок.
2. Инициализация Git-репозитория и push в `QLord-ops/traidingbot`.
3. Улучшение backtest:
   - исторические данные за несколько лет с пагинацией и локальным кэшем;
   - funding fees;
   - maker/taker fees как параметры;
   - slippage/spread model;
   - запрет look-ahead;
   - корректная обработка свечи, где одновременно затронуты SL и TP, с консервативным сценарием;
   - equity curve и drawdown curve;
   - monthly returns;
   - trade expectancy, average R, consecutive losses;
   - out-of-sample и walk-forward.
4. Веб-интерфейс:
   - сохранение профилей стратегии;
   - выбор диапазона дат;
   - графики equity, drawdown и цены с входами/выходами;
   - сравнение нескольких параметров;
   - понятные предупреждения о переоптимизации;
   - русский интерфейс.
5. Testnet engine:
   - async loop/WebSocket;
   - server time sync;
   - exchange filters and precision;
   - idempotent client order IDs;
   - reconciliation с реальной позицией и ордерами биржи;
   - isolated margin и leverage;
   - one-way/hedge mode validation;
   - atomic entry + protective SL/TP flow;
   - emergency close;
   - daily loss lockout;
   - max trades/open positions;
   - restart recovery;
   - structured logs and SQLite/PostgreSQL state.
6. Telegram уведомления и аварийная кнопка.
7. Docker Compose, `.env.example`, healthcheck и инструкции запуска.
8. Только после 100–200 Testnet сделок и проверенной статистики обсуждать live mode.

## Критерии Testnet MVP

- Все тесты проходят.
- Ни одного секрета в Git.
- Веб-панель запускается одной командой или через Docker Compose.
- Backtest воспроизводим.
- Testnet-позиция не может остаться без SL.
- Повторный запуск не создаёт дублирующую позицию.
- При потере соединения состояние восстанавливается с Binance.
- Есть кнопка STOP и аварийное закрытие.
- Live mode отсутствует либо жёстко заблокирован.
